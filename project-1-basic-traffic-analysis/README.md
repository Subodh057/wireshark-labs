[02:00, 4/9/2026] Subodh: # Basic Network Traffic Analysis using Wireshark

## Objective
Capture and analyze basic network traffic using Wireshark to understand how common protocols such as DNS, HTTP, HTTPS, and ICMP work.

## Tools Used
- Wireshark
- Linux Terminal
- Web Browser

## Traffic Generated
- Visited websites such as Google, GitHub, and YouTube
- Accessed an HTTP site for unencrypted web traffic
- Used ping to generate ICMP traffic

## Filters Used

### DNS
dns

### HTTP
http

### HTTPS
tcp.port == 443

### ICMP
icmp

## Key Observations

### DNS
Observed domain resolution requests and responses, showing how websites are translated into IP addresses.

### HTTP
Observed readable web requests such as GET requests and host headers.

### HTTPS
Observed encrypted traffic over port 443. Payload contents were not readable.

### ICMP
Observed echo request and echo reply packets generated by ping.

## Learning Outcomes
- Understood how DNS translates domain names to IP addresses
- Learned the difference between HTTP and HTTPS visibility
- Identified ICMP traffic used for connectivity testing
- Practiced using Wireshark filters to isolate protocol traffic
